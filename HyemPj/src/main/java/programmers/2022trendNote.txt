1.프론트엔드
- React, Vue.js, Svelte
- 많은 기업들이 리액트, 뷰를 '그대로'사용하기 보단 새로운 '프레임워크'를 사용하게 되지 않을까 예상
- React : 재사용 가능한 컴포넌트를 통해 인터페이스를 구축하기 위한 프론트엔드 개발전용 자바스크립트 라이브러리
	+ 특징 : 1) 재사용이 가능한 UI컴포넌트
			2) JSX 지원
			3) Vertual DOM
			4) 단방향 데이터 바인딩
	+ 리액트를 사용하는 이유 : 1) 강력한 커뮤니티 지원(페이스북)
						2) SEO 친화적
						3) 유연성 및 호환성
	(Next.js, Gatsby, Remix, Redwood..)
- Vue 기반의 Nuxt
- Svelte : 리액트와 뷰 등 기존 언어를 보완하고자 2016년에 출시됨.
	+ 특징 : 1) 낮은 러닝 커브, 코드량 감소
			2) 작은 크기의 번들
			3) 스벨트의 빠른 속도
			4) 가상 DOM 비교 처리가 없음
	+ React와 비교 : 1) 스벨트가 더 뛰어난 성능을 보여준다
					2) 스벨트는 가상 DOM 비교 처리가 없다
					3) 스벨트의 반응성이 더 뛰어난다
					4) 스벨트는 다른 방식으로 컴포넌트를 처리한다
					5) 스벨트를 사용하는 데 많은 외부 라이브러리가 필요하지 않다
					6) 스벨트는 더 가볍다
					7) 스벨트를 사용하면 개발이 빨라진다
					8) 스벨트가 더 배우기 쉽다
	(SvelteKit..)
